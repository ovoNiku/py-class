### 如何在实际中解决问题

我们把代码问题分为两种, 一种是系统错误, 一种是逻辑错误

系统错误就是系统有明确的提示告诉你出错了, 并给出了相关的报错信息, 请务必认真阅读相关的信息

另一种是逻辑错误, 虽然系统没有任何报错, 但是产生的结果并不符合你的预期

我们分开讨论

#### 系统错误

一般系统错误会有一个堆栈信息, 里面会给出报错的位置, 以及报错的上下文(从哪一行调用过去的)

你应该检查对应的代码的错误(尤其是你写的那一行的)

另外代码功能应该逐个添加, 因为如果一份代码有十几份错误, 你就很难解决和验证错误的原因

需要理解错误信息的含义

很多人看到错误, 丢到 百度 (maybe google) 会天真的认为搜到一个答案, 照着修改就会成功

也有人会说使用搜索引擎是一个程序员的基本素质, 是一种必要能力

实际上搜索引擎经常不会返回你满意的结果, 或者说无法直接解决你的问题

你应该尝试先把问题缩小成一个更小的场景, 比如之前比较典型的问题: 

`请问我在 flask 中写了一个route方法 add, 调用 add 方法后 报错 'tuple' object has no attribute 'append', 请问我该怎么办?`

实际上这可能和 add 方法并没有什么关系, 问题的核心是对一个 data_list 进行 append 的时候报错了, 这时候 debug 会发现, 声明的并不是 list 类型

为什么会有人说工作代码不能咨询别人, 注定要自己解决, 实际上这是错误的, 工作代码也完全可以抽成单独的代码 

总之要尝试理解错误, 并且想方设法的验证问题, 拆解问题, 并解决问题

#### 逻辑错误

逻辑错误会更麻烦一些, 但其实麻烦在你需要自己梳理逻辑, 找到逻辑漏洞, 打印相关日志查看数据的情况来验证自己的想法

尤其是在一些多线程、异步回调等特殊的程序场景问题很容易变得复杂

讲清楚很重要, 这种问题往往你并不需要咨询别人, 只需要自我对话就可以解决

小黄鸭调试法, 软件工程中使用的调试代码方法之一, 说的就是这种情况
